# Documentation Directory

All documentation is written in Markdown. Twelve files are included:

| File | Type | Purpose |
|------|------|---------|
| `methodology.md` | Technical | I-O model math — matrix notation, formulas, assumptions, validation approach |
| `technical_memo.md` | R&D Report | Internal memo with 11 sections; detailed enough for a qualified analyst to reproduce results |
| `policy_brief.md` | Executive | Plain-English two-pager for non-technical readers; explains direct/indirect/induced in accessible language |
| `data_dictionary.md` | Reference | Full schema documentation for all 5 sample CSV files |
| `procurement_evidence.md` | Procurement | Capability statement, PWS-to-evidence mapping table, resume and capability statement snippets |
| `assumptions_and_limitations.md` | Compliance | Explicit, direct statement of all model limitations and appropriate/inappropriate use cases |
| `validation_report.md` | QA | Data validation status, unit test status, sanity checks, known unresolved issues |
| `architecture.md` | Technical | System overview, Mermaid data-flow diagram, module responsibilities, future integration points |
| `accessibility_checklist.md` | Compliance | Section 508 design intentions, items checked, known gaps, disclaimer of non-compliance |
| `work_plan_template.md` | Template | Reusable work plan structure for future project extensions |
| `technical_memo_template.md` | Template | Reusable technical deliverable memo structure |
| `monthly_progress_report_template.md` | Template | Reusable monthly progress report structure |
